# 大分市 通称・公称 対照データ変換指示書(Gemini 用)

- 文書バージョン: 1.0
- 作成日: 2026-07-09
- 対象工程: データ整備(対照資料 → CSV 変換)
- 実行担当: Gemini(マルチモーダル読み取り)
- 後工程: Claude Code(CSV → GeoJSON 変換・座標付与)
- 関連文書: `docs/spec-mvp.md`(アプリ本体仕様)

---

## 1. あなた(Gemini)への依頼内容

これから渡す大分市の町名に関する資料(PDF・スキャン画像・表など)を読み取り、
「通称町名」と「公称町名」の対応関係を **1 行 1 対応の CSV** に書き起こしてください。

このデータは、地図アプリで通称⇔公称のラベルを切り替えるために使用します。
**正確さが最優先**です。読み取れない・確信が持てない箇所は、推測で埋めずに
その旨を明示してください(ルールは §5 参照)。

## 2. 入力資料について

- 資料は複数回に分けて渡す場合があります。1 回の依頼で渡した資料の範囲のみを処理してください
- 資料の種類の想定: 大分市の住居表示関連資料、町名対照表、広報資料、法務局関連資料など
- スキャン品質が低い場合があります。判読不能な文字は §5 のルールに従ってください

## 3. 出力 CSV の仕様

### 3.1 基本ルール

- 文字コード: UTF-8
- 1 行目はヘッダー行
- 区切り: カンマ。フィールド内にカンマ・改行が含まれる場合はダブルクォートで囲む
- CSV 本体のみをコードブロックで出力し、前置き・解説文は最小限にする

### 3.2 列定義

| 列名 | 必須 | 説明 |
|---|---|---|
| seq | ○ | 連番(1 から。ゼロ埋め不要。最終的な id 採番は後工程で行う) |
| tsusho | ○ | 通称町名(丁目まで含む場合はそのまま記載) |
| kosho | ○ | 公称町名(「大分市」は含めない。大字・字を含む正式表記) |
| source_doc | ○ | 資料名またはファイル名 |
| source_page | ○ | 該当ページ番号(不明なら空欄) |
| confidence | ○ | high / medium / low(§5 の基準で判定) |
| note | - | 判読困難箇所、併記されていた補足、疑義など |

### 3.3 出力例

```csv
seq,tsusho,kosho,source_doc,source_page,confidence,note
1,中央町一丁目,大字大分字○○,大分市町名対照表.pdf,3,high,
2,府内町二丁目,大字荷揚字△△,大分市町名対照表.pdf,3,medium,字名の2文字目がかすれており「△△」は要確認
3,元町,元町,大分市町名対照表.pdf,4,high,通称と公称が同一
```

### 3.4 座標について(重要)

**座標(緯度・経度)は出力に含めないでください。**
資料に座標が記載されていることは通常なく、推定で座標を生成すると誤データの
混入源になります。座標付与は後工程で Claude Code が
国土地理院のジオコーディング API 等を用いて機械的に行います。

例外: 資料自体に座標が明記されている場合のみ、note 列に
「座標記載あり: 33.xxxx, 131.xxxx」の形式で転記してください。

## 4. 表記の正規化ルール

読み取った文字列は、以下のルールで正規化してから出力してください。

1. **数字**: 丁目・番地の数字は漢数字のまま転記する(「一丁目」を「1丁目」に変換しない)。資料が算用数字ならそのまま
2. **大字・字**: 資料の表記どおりに残す。「大字」「字」を省略・補完しない。資料側で省略されている場合は note に「資料上『大字』表記なし」と記録
3. **空白**: 町名内の全角・半角スペースはすべて除去する
4. **旧字体・異体字**: 資料の字体をそのまま転記し、note に「旧字体: 〇(新字体: △)」と記録。勝手に新字体へ置換しない
5. **繰り返し記号**: 「々」「ヶ」「ケ」「が」等は資料の表記どおり。判別しづらい場合(ヶ/ケ など)は note に記録
6. **同一名称**: 通称と公称が同じ町は、両列に同じ値を入れる(空欄や「同上」にしない)
7. **1 対多の対応**: 1 つの通称に複数の公称が対応する場合(またはその逆)は、**組み合わせごとに行を分ける**。note に「1対多: ○○に対し公称△件」と記録

## 5. 確信度(confidence)と読み取り品質のルール

- **high**: 文字が明瞭で、対応関係も資料上明確
- **medium**: 一部の文字がかすれ・低解像度だが、文脈からほぼ確実に判読できる
- **low**: 判読に推測が入っている。note に必ず「どの文字が不明か」を記載

絶対ルール:

- 判読できない文字は「■」で置き換え、confidence を low にする。**それらしい漢字で埋めない**
- 資料に存在しない行を補完・創作しない(「この地区なら他にこの町もあるはず」という推測は禁止)
- 資料の対応関係に疑問がある場合(誤植の可能性など)も、まず資料どおりに転記し、疑義は note に書く

## 6. 出力の最後に付けるサマリー

CSV の後に、以下の 4 点だけを簡潔に報告してください。

1. 処理した資料名とページ範囲
2. 出力行数
3. confidence 別の件数(high / medium / low)
4. 特に人間の確認が必要な行(low の seq 番号一覧)

## 7. 後工程への引き継ぎ(参考情報)

あなたの出力 CSV は、この後 Claude Code が以下を行います。

- 国土地理院 API 等による座標付与(ジオコーディング)
- `chomei.geojson` への変換(スキーマ: id / tsusho / kosho / note)
- 人間による low 行のレビュー結果の反映

そのため、**きれいな整形よりも「資料に忠実+疑義の明示」を優先**してください。
