# 大分市 町名切り替え地図アプリ(MVP)仕様書

- 文書バージョン: 1.0
- 作成日: 2026-07-09
- 対象フェーズ: MVP(町名ラベル切り替えのみ)
- 実装担当: Claude Code
- 公開形態: GitHub パブリックリポジトリ + GitHub Pages

---

## 1. 概要

大分市内の町名について、日常的に使われる「通称町名」と登記・公簿上の「公称町名」を、
地図上のラベルとしてボタン1つで切り替えて表示する静的 Web アプリを作成する。

サーバーサイド処理は一切持たず、静的ファイルのみで完結させる(GitHub Pages で配信)。

## 2. スコープ

### 2.1 MVP に含むもの

1. 大分市中心部を初期表示する地図(地理院タイル)
2. 町名ラベルの地図上表示(GeoJSON のポイントデータに基づく)
3. 画面上のトグルボタンによる「通称 ⇔ 公称」の即時切り替え
4. 現在の表示モード(通称 / 公称)が一目で分かる UI
5. 地図の基本操作(パン・ズーム、ピンチ操作対応)
6. スマートフォン・PC 両対応のレスポンシブ表示
7. サンプルデータ(10〜20 地点程度)での動作確認

### 2.2 MVP に含まないもの(第2弾以降)

- GPS / 現在地表示(**本アプリでは今後も不要**)
- 町域ポリゴンの塗り分け表示
- 町名検索機能
- 通称・公称の由来などの解説表示
- 全町名の完全なデータ整備(MVP はサンプルデータで可)

## 3. 技術構成

| 項目 | 採用技術 | 備考 |
|---|---|---|
| 地図ライブラリ | MapLibre GL JS(CDN 読み込み) | ラベルのプロパティ切り替えが得意 |
| ベース地図 | 国土地理院 淡色地図タイル | `https://cyberjapandata.gsi.go.jp/xyz/pale/{z}/{x}/{y}.png` |
| データ形式 | GeoJSON(静的ファイル) | fetch で読み込み |
| 言語 | HTML / CSS / JavaScript(Vanilla) | ビルドツール不使用、フレームワーク不使用 |
| ホスティング | GitHub Pages | main ブランチ push で自動デプロイ(Actions) |

依存パッケージのインストールを不要とし、`index.html` をブラウザで開くだけでも
動作すること(ローカル確認は `python -m http.server` 等の簡易サーバーを想定)。

## 5. データ仕様(chomei.geojson)

FeatureCollection 形式。1 Feature = 1 町名地点(ラベル表示位置)。

```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "geometry": {
        "type": "Point",
        "coordinates": [131.6096, 33.2396]
      },
      "properties": {
        "id": "0001",
        "tsusho": "○○町",
        "kosho": "大字△△字□□",
        "note": ""
      }
    }
  ]
}
```

### プロパティ定義

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| id | string | ○ | 4桁ゼロ埋めの連番 |
| tsusho | string | ○ | 通称町名 |
| kosho | string | ○ | 公称町名 |
| note | string | - | 備考(出典メモ等。MVP では表示しない) |

- 座標系は WGS84(経度, 緯度 の順)
- 通称と公称が同一の町は tsusho と kosho に同じ値を入れる
- MVP 用サンプルデータは大分市中心部の実在町名を 10〜20 件、
  座標はおおよその町域中心でよい(精度は問わない)

## 6. 画面仕様

### 6.1 レイアウト

- 地図を画面全体(100vw × 100vh)に表示
- 左上: アプリタイトル「大分市 町名マップ」(小さく、地図操作を妨げない)
- 上部中央または右上: 表示切り替えトグルボタン(後述)
- 右下: 出典表示「地理院タイル」(地図ライブラリの attribution 機能を使用)

### 6.2 トグルボタン

- 「通称」「公称」の 2 択が並んだセグメント型トグル(iOS 風)
- 現在選択中のモードが背景色反転等で明確に分かること
- タップ / クリックで即時切り替え(アニメーション・確認ダイアログ不要)
- スマホでの押しやすさを考慮し、タップ領域は高さ 44px 以上
- 初期表示は「通称」モード

### 6.3 町名ラベル

- MapLibre の symbol レイヤーで表示
- フォントサイズはズームレベルに応じて 12〜16px 程度で可変
- 文字色: 濃紺系、白フチ(halo)付きで地図上でも可読性を確保
- 切り替えは `text-field` の参照プロパティを `tsusho` ⇔ `kosho` に
  差し替える方式(`setLayoutProperty` を使用。レイヤー再生成はしない)
- ラベル同士の重なりは MapLibre の自動間引きに任せる

### 6.4 初期表示位置

- 中心座標: 大分駅付近(経度 131.606, 緯度 33.233 目安)
- 初期ズーム: 14
- 最小ズーム 11 / 最大ズーム 18

## 7. 動作仕様

1. ページ読み込み時に `chomei.geojson` を fetch で取得
2. 取得失敗時は画面上部にエラーメッセージを表示(alert は使わない)
3. トグル操作時、状態変数 `currentMode`("tsusho" | "kosho")を更新し、
   symbol レイヤーの `text-field` を切り替える
4. ページリロード時は常に「通称」モードに戻る(状態の永続化は不要)

## 8. デプロイ仕様

- GitHub Actions で main ブランチへの push を契機に `src/` 配下を
  GitHub Pages へデプロイする(actions/deploy-pages を使用)
- リポジトリの Pages 設定は「GitHub Actions」ソースを前提とする
