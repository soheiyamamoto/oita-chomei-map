# 大分市 住居表示旧新対照簿 変換指示書 v2(Gemini 用)

- 文書バージョン: 2.0(v1: `data-conversion-gemini.md` を全面改訂)
- 作成日: 2026-07-10
- 対象工程: データ整備(対照簿 PDF/抽出テキスト → CSV 変換)
- 実行担当: Gemini
- 後工程: Claude Code(CSV 検証 → lookup.json 統合)
- 関連文書: `docs/spec-v2-address-lookup.md`

## 1. あなた(Gemini)への依頼内容

これから渡す大分市の「住居表示旧新対照簿」(PDF またはそこから抽出したテキスト)を
読み取り、**1 行 1 対応の CSV** に書き起こしてください。

このデータは「旧住所や通称しか分からない人が現在の正式住所を調べる」検索ツールに
使用します。**正確さが最優先**です。読み取れない・確信が持てない箇所は、
推測で埋めずにその旨を明示してください(§5)。

## 2. 資料の構造(重要)

対照簿は次の 3 層構造です。1 行に 3 つが並びます。

| 層 | 内容 | 例 |
|---|---|---|
| 旧住所 | 大字+番地 | 大字木上154番地の3 |
| 通称 | 団地名・建物名等(**空欄の行あり**) | カームタウン木ノ上A－10 |
| 新住所 | 新町名+街区番号+住居番号 | 木上台一丁目 3 9 |

- 新住所の数字 2 つは「街区番号」「住居番号」の順
- 列ヘッダー(「大字・番地」「通称」「町名」「番号」等)がページごとに繰り返される
  ことがあるが、これはデータ行ではないので出力しない
- 大字ごとにセクションが分かれ、ページ番号が振り直されることがある

## 3. 出力 CSV の仕様

### 3.1 基本ルール

- 文字コード: UTF-8、1 行目はヘッダー行
- 区切りはカンマ。フィールド内にカンマ・改行が含まれる場合はダブルクォートで囲む
- CSV 本体のみをコードブロックで出力し、前後の解説は最小限にする
- 行数が多い場合は分割して出力してよい(その場合、続きにもヘッダーは付けない)

### 3.2 列定義

| 列名 | 必須 | 説明 |
|---|---|---|
| seq | ○ | 連番(1 から) |
| old_address | ○ | 旧住所(大字+番地)。資料の表記どおり |
| alias | - | 通称(団地名・建物名)。資料で空欄なら空欄のまま |
| new_town | ○ | 新町名(例: 木上台一丁目) |
| block | ○ | 街区番号(数字のみ) |
| house | ○ | 住居番号(数字のみ) |
| source_doc | ○ | 資料名またはファイル名 |
| source_page | ○ | 該当ページ番号。セクションごとに振り直されている場合は
「木上-3」のように大字名を付けて区別する。不明なら空欄 |
| confidence | ○ | high / medium / low(§5 の基準) |
| note | - | 判読困難箇所、疑義、重複の記録など |

### 3.3 出力例

```csv
seq,old_address,alias,new_town,block,house,source_doc,source_page,confidence,note
1,大字木上154番地の3,カームタウン木ノ上A－10,木上台一丁目,3,9,kinouedaitaisyoubo.pdf,木上-1,high,
2,大字木上201番地の109,,木上台一丁目,2,16,kinouedaitaisyoubo.pdf,木上-2,high,資料上通称の記載なし
3,大字市■■番地の2,カームタウン木ノ上D－5,木上台二丁目,1,7,kinouedaitaisyoubo.pdf,市-1,low,番地の2文字が判読不能
```

## 4. 正規化・整形ルール

1. **表記は資料に忠実に**: 数字の全角/半角、「番地の」等の表記は資料どおり転記する
   (検索用の正規化は後工程のプログラムで行うため、ここでは変換しない)
2. **空白**: 各フィールド内の前後空白は除去。住所文字列内部の空白も除去
3. **通称の空欄**: 資料で通称欄が空欄の行は alias を空欄にし、note に
   「資料上通称の記載なし」と記す。これは low ではなく high でよい
   (空欄であること自体が資料の内容であるため)
4. **完全重複行**: 全項目が同一の行が資料に複数回現れた場合
   (改ページやセル結合による重複抽出が起きやすい)、**1 行だけ出力**し、
   note に「資料上N回出現」と記録する
5. **旧字体・異体字・繰り返し記号**: 資料の字体のまま転記し、note に記録。
   勝手に置換しない
6. **1つの旧住所に複数の新住所**(またはその逆)が対応する場合は、
   組み合わせごとに行を分け、note に「1対多」と記録する

## 5. 確信度(confidence)と品質ルール

- **high**: 明瞭に判読でき、3 層の対応も資料上明確
- **medium**: 一部かすれ等があるが文脈からほぼ確実に判読できる
- **low**: 判読に推測が入っている。note に「どの文字が不明か」を必ず記載

絶対ルール:

- 判読できない文字は「■」で置き換え、confidence を low にする。
  **それらしい文字で埋めない**
- 資料に存在しない行を補完・創作しない
- 行の対応関係(どの旧住所がどの新住所と同じ行か)に崩れの疑いがある場合は、
  その範囲の行をすべて low とし、note に「行対応に疑義」と記す
- 抽出テキストを渡された場合も同じルールを適用する。テキストの乱れで
  対応が確定できない行は、埋めずに low+note で報告する

## 6. 出力の最後に付けるサマリー

1. 処理した資料名と範囲(大字別の行数内訳)
2. 出力行数と、重複除外した行数
3. confidence 別の件数(high / medium / low)
4. 人間の確認が必要な行(low の seq 一覧、■ を含む行の seq 一覧)
5. 通称が空欄だった行数

## 7. 後工程への引き継ぎ(参考情報)

あなたの出力 CSV は、この後 Claude Code が検証(列数・数字妥当性・重複)を行い、
全地区統合の `lookup.json` に変換します。表記ゆれの正規化(全半角・漢数字など)も
後工程で機械的に行うため、**きれいな整形よりも「資料に忠実+疑義の明示」を
優先**してください。
