## 1. 背景と目的の転換

v1(MVP)は「通称⇔公称の地図ラベル切り替え」を目的としていたが、
本来解決したい課題は「**通称や旧住所で書かれた郵便物が届かない**」事象であると
明確になった。大分市では住居表示の実施により住所が順次変更されており
(令和7年11月時点で38地区)、旧住所(大字+番地)や通称(団地名・建物名)しか
分からない状態から**現在の正式住所を確認できる検索ツール**へ転換する。

- コア機能: 住所検索(旧住所・通称・新住所のいずれからでも検索可能)
- 補助機能: 該当地区の位置を地図で表示(v1の地図実装を流用)

## 2. データソース

大分市公式サイト「住居表示旧新対照簿」(38地区分のPDF)。
各対照簿は概ね次の3層構造を持つ:

1. 旧住所: 大字+番地(例: 大字木上154番地の3)
2. 通称: 団地名・建物名等(例: カームタウン木ノ上A－10)※空欄の行もある
3. 新住所: 新町名+街区番号+住居番号(例: 木上台一丁目 3番 9号)

データ整備工程は `docs/data-conversion-gemini-v2.md` に従い、
地区別CSV(`data/sources/`)→統合JSON(`src/data/lookup.json`)の順に構築する。

## 3. 機能仕様

### 3.1 検索機能(コア)

- 単一の検索ボックスに自由入力。入力対象は旧住所・通称・新住所のいずれでもよい
- 部分一致検索(例: 「木上154」「カームタウン」「木上台一丁目」いずれもヒット)
- 検索実行はインクリメンタル(入力2文字以上で自動検索、debounce 300ms)
- 表記ゆれの正規化(検索時に入力とデータ双方へ適用):
  - 全角英数字・記号 ⇔ 半角の同一視
  - 漢数字 ⇔ 算用数字の同一視(一丁目=1丁目、百五十四=154)
  - 「番地の」「番地」「−」「-」「の」の同一視(154番地の3 = 154-3)
  - 空白の除去、カタカナの全半角同一視
- 結果は一覧表示。各行に以下を表示:
  - 現在の正式住所(新町名+街区番号+住居番号)を最も目立つ形で
  - 旧住所、通称(あれば)
  - 地区名、住居表示の実施日
- ヒット件数が多い場合は上位50件+件数表示(絞り込みを促すメッセージ)
- 0件時: 「該当が見つかりません」+ 未収録地区の可能性と市の窓口
  (市民協働推進課 097-537-7250)を案内

### 3.2 地図表示(補助)

- 検索結果の行を選択すると、該当**地区**のおおよその位置に地図を移動しマーカー表示
- 位置は地区代表点(地区ごとに1点)でよい。番地単位のジオコーディングは行わない
- v1のMapLibre実装・地理院タイルをそのまま流用
- 画面構成: 上部に検索ボックス+結果一覧、下部(またはPC時は右側)に地図

### 3.3 免責・出典表示

- ページ下部に常設: 「本ツールは大分市公開の住居表示旧新対照簿を基にした
  参考情報です。正式な確認は大分市市民協働推進課へお問い合わせください」
- データ出典として市サイトの対照簿ページへのリンクを表示
- README に市サイトコンテンツの利用条件の確認結果を記載すること(実装前に要確認)

## 4. データ仕様(lookup.json)

```json
{
  "meta": {
    "generated": "2026-07-10",
    "source": "大分市 住居表示旧新対照簿",
    "districts": 38
  },
  "records": [
    {
      "id": "kinoue-0001",
      "district": "カームタウン木ノ上地区",
      "old": "大字木上154番地の3",
      "alias": "カームタウン木ノ上A－10",
      "new_town": "木上台一丁目",
      "block": "3",
      "house": "9",
      "date": "2021-01-16",
      "conf": "high"
    }
  ]
}
```

- alias は空文字を許容(通称のない行)
- 検索用の正規化済み文字列は実行時生成でもビルド時付与でもよい(実装判断に委ねる)
- confidence が low の行は結果表示時に「要確認」バッジを付ける
- 想定規模: 全38地区で数千〜数万行。JSONが3MBを超える場合は
  地区別分割+遅延読み込みを検討(初回は単一ファイルで実装し計測して判断)

## 5. 画面仕様

- v1の全画面地図レイアウトから、検索ファーストのレイアウトへ変更
- モバイル: 検索ボックスを画面上部固定、結果一覧をスクロール、地図は結果選択時に
  下半分へ表示
- PC: 左に検索+結果一覧(幅40%程度)、右に地図
- 検索ボックスは placeholder で入力例を提示
  (例: 「大字木上154」「カームタウン」など)
- 高さ指定は `100dvh`(`100vh` フォールバック付き)を踏襲

## 6. v1からの変更点まとめ

| 項目 | v1 (MVP) | v2 |
|---|---|---|
| コア機能 | 地図ラベルの通称⇔公称切り替え | 住所検索(旧→新の引き当て) |
| データ | 町名ポイント15件(サンプル) | 対照簿全行(番地レベル) |
| 地図の役割 | 主役 | 補助(地区位置の表示) |
| トグルUI | あり | 廃止(検索UIへ置換) |
| ジオコーディング | 予定あり | 地区代表点のみ(番地単位は不要) |

- v1のトグルUIと `chomei.geojson` は削除せず、`docs/` に経緯を記録した上で
  リポジトリ履歴に残す(READMEからの導線は外す)
