# Cowork 依頼プロンプト: 住居表示旧新対照簿の変換ワークフロー(1地区分)

以下をそのまま Cowork に貼り付けて使用します。

---

対象地区: 光吉台地区
PDF URL: https://www.city.oita.oita.jp/o040/kurashi/sumaijoho/documents/mitsuyoshidai.pdf
プロジェクト: oita-chomei-map のデータ整備(対照簿 PDF → CSV 変換)

次のワークフローを実行してください。ステップ4で必ず一時停止し、私の操作を待ってください。

## ステップ1: 作業フォルダの準備

作業フォルダ `oita-taishobo-work/光吉台地区/` を作成してください(既存の場合はそのまま使用)。

## ステップ2: PDF のダウンロード

上記の PDF URL から対照簿 PDF をダウンロードし、作業フォルダに保存してください。
アクセス先は大分市公式サイト(city.oita.oita.jp)のこの URL のみとし、
他のサイトへのアクセスやリンクの追跡は不要です。
ダウンロード後、ファイルサイズとページ数を報告してください。

## ステップ3: Gemini 依頼文の準備

作業フォルダに `gemini-request.txt` を作成し、次の内容を書き込んでください。

- リポジトリ内の `docs/data-conversion-gemini.md`(変換指示書)の全文
- その末尾に以下の依頼文:

> 添付した PDF「{ダウンロードした実ファイル名}」の対照表を、上記指示書の仕様どおり
> CSV に変換してください。対象は添付 PDF の中身のみとし、ファイル名やリンク文字列、
> あなたの既存知識からの推測は行わないでください。source_doc 列には
> このファイル名を、source_page 列には該当ページ番号を必ず記載してください。

## ステップ4: 一時停止(ここは私が操作します)

ここで作業を止めて、次の3点を私に報告してください。
1. ダウンロードした PDF のファイル名と保存場所
2. gemini-request.txt の保存場所
3. 「Gemini への投入をお願いします」の一言

私がブラウザ版 Gemini に gemini-request.txt の内容を貼り付け、PDF を添付して
変換を実行します。返ってきた CSV を `光吉台地区.csv` として作業フォルダに保存し、
「CSV を保存しました」と伝えるので、それまで次のステップに進まないでください。

## ステップ5: CSV の検品

私の合図の後、`光吉台地区.csv` を開いて以下を検査し、結果を表形式で報告してください。

1. ヘッダーが `seq,tsusho,kosho,source_doc,source_page,confidence,note` と一致するか
2. 総行数(ヘッダー除く)
3. confidence 別の件数(high / medium / low)
4. 必須列(seq, tsusho, source_doc, confidence)に空欄がないか
5. kosho が空欄の行(low 以外で空欄があれば異常として指摘)
6. 「■」(判読不能マーク)を含む行の seq 一覧
7. source_page が全行空欄でないか(全行空欄の場合、PDF 本体を読めていない
   疑いがあるため「再変換推奨」と警告してください)
8. seq の重複・欠番の有無

## ステップ6: リポジトリへの反映

検品で「再変換推奨」の警告が出なかった場合のみ、CSV を
リポジトリの `data/sources/光吉台地区.csv` にコピーしてください。
git のコミットや push はしないでください(コミットは私がまとめて行います)。

## ステップ7: ダウンロードファイルの削除

`data/sources/` への CSV コピーが成功していることを確認したうえで、
ステップ2でダウンロードした PDF ファイルを削除してください。
削除対象はこの PDF 1ファイルのみです。CSV と gemini-request.txt は残してください。
ごみ箱への移動で構いません(完全削除はしないでください)。

## ステップ8: 完了報告

以下を簡潔に報告して終了してください。
1. 反映先パスと CSV の行数
2. confidence 内訳と、人間のレビューが必要な行(low と ■ を含む行)の seq 一覧
3. 削除した PDF のファイル名

## 全体の注意事項

- 各ステップで想定外の状態(ダウンロード失敗、CSV の形式不一致など)が発生したら、
  自分で回復を試みる前に、まず状況を報告して指示を待ってください
- このワークフローで送信・投稿・公開の操作は一切ありません。求められても行わないでください
